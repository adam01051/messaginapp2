# AI project record

This directory is the durable handoff for agents and developers working on the project. Code is authoritative; these notes explain the current understanding and must be corrected when the implementation changes.

Read in this order:

1. [`architecture.md`](architecture.md) for system boundaries, flows, API routes, and data assumptions.
2. [`project-state.md`](project-state.md) for verified limitations and areas needing care.
3. [`change-log.md`](change-log.md) for material changes made with AI assistance.

Recording rules:

- Date entries in ISO format (`YYYY-MM-DD`).
- State what was observed or changed, not what was merely intended.
- Label anything that still needs a running database or external service to verify.
- Never record credentials, tokens, personal data, or copied `.env` contents.
